🚀 Getting Started

AWKit (Antigravity Kit) is an advanced tool for project orchestration, code generation, and AI agent coordination. Your project has been initialized with following essential files:

  • .project-identity: Contains project metadata (Name, Tech Stack, Bundle ID). Edit this to change project constraints.
  • CODEBASE.md: The living documentation of your codebase architecture. Run /codebase-sync in your AI Chat to update it automatically.
  • AGENTS.md & CLAUDE.md: Router files that point your AI IDE (Cline, Roo, Cursor, Claude Code) to the central rulebook.
  • .symphony/: Tracks tasks and progress via Symphony Orchestrator.

⚡ Core Workflows

Use these slash commands in your AI chat to trigger expert behaviors:

Command Description
/brainstorm Ideate and research new features. Outputs a Spec/Brief document.
/plan Design architecture and technical approach before coding.
/code Write production code based on a completed Spec.
/debug Systematic debugging process (diagnose → verify → fix).
/next Ask context-aware "What should I do next?" (Syncs with Symphony & NeuralMemory).
/recap Generate a summary of the project and current state.
/gitnexus Explore codebase via Knowledge Graph.

🧠 GitNexus Code Intelligence

AWKit uses GitNexus to understand the entire architecture, blast radiuses, and execution flows. It builds a graph of your code.

Important CLI Commands:

# Re-analyze code and update graph (Run this after major changes)
npx gitnexus analyze

# Force full update including embeddings
npx gitnexus analyze --embeddings

AI Agents: Always use GitNexus tools like gitnexus_impact before refactoring, and gitnexus_query to explore.

📋 Symphony Task Tracking

AWKit uses Symphony to track what's currently being worked on. Symphony serves as the brain for the context boundaries.

# View current tasks
symphony task list

# Add a manual task
symphony task add "Update UI configuration"

# Check system status
symphony status

🤖 The 7-Gate Autonomous System

Complex tasks requested in AWKit follow a strict, reliable process known as the 7-Gate pipeline:

  1. Gate 1 (Spec): Create requirements / BRIEF.md
  2. Gate 1.5 (Module Spec): Drill down into screen flows and business rules.
  3. Gate 2 (Architecture): Technical DB/API design.
  4. Gate 2.5 (Visual): Pencil MCP or reference screenshots.
  5. Gate 3 (Tasks): Break into Symphony tickets.
  6. Gate 4 (Execution): 3-Phase approach (Infrastructure → UI Shell → Logic) with User Checkpoints.
  7. Gate 5 (Verification): Code review, testing, and Git operations.

🔌 Plugins & Integrations

  • Trello Sync: Run awkit trello setup and awkit trello sync to connect the project to Trello. Automatically updates checklists based on Symphony.
  • Neural Memory: Contextual long-term memory for AI agents to remember edge cases, architecture decisions, and logic across sessions.
  • Telegram Notify: Send build output or deployment updates to Telegram via awkit tg send "Message".
